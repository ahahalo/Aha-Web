## ❓ Frequently Asked Questions (FAQ) 

*1. Is it free?*
Yes, this application is completely free to use. It was developed to provide users with an easier way to manage their frequency programs and customize their AHA Halo experience without additional costs or downloads.

*2. Will it damage my AHA Halo device?*
No, the app does not damage the hardware. It communicates with the device using the standard protocols that the original software uses. It simply sends data values and frequencies to the internal slots of the device. However, always ensure you are using a stable USB connection when writing data to prevent communication errors.

*5. Why the need for another app?*
The original software was created in Xojo. Xojo is an application builder, but it’s a very poor one since it simply performs very poorly on the computer (i.e., it crashes and is extremely slow). That .sqlite file contains all the frequencies. What we did is we checked the logs generated by the official software, reverse engineered them, and intercepted all USB connections the computer makes with the Aha Halo. This way, we can read and write data.

Interface: Serial connection via USB (often using a CH340 or CP210x chip). Baud rate: 115200 bps. Command structure: Uses ASCII strings that begin with a colon (:) and end with a period (.) or a comma (,), followed by a Newline (\r\n).  The Halo has temporary buffer memory (RAM) for on-screen display and permanent Flash memory for storage. Write Protection: Crucial is the command :w09=0 to disable write protection and :w09=1 to re-enable it. Without this step, storage is not permanent. Aha Halo has a EEPROM. Burning: The command :w84=255 activates the physical writing process to the chip, which takes about 3 to 4 seconds. Naming (:n): ASCII text linked to the slot number.
