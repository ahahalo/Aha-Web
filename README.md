## ❓ Frequently Asked Questions (FAQ) 

*1. Is it free?*
Yes, this application is completely free to use. It was developed to provide users with an easier way to manage their frequency programs and customize their Aha experience without additional costs or downloads.

*2. Will it damage my Aha device?*
No, the app does not damage the hardware. It communicates with the device using the standard protocols that the original software uses. It simply sends data values and frequencies to the internal slots of the device. However, always ensure you are using a stable USB connection when writing data to prevent communication errors.

*3. It’s not working, what should I do?*
If you encounter issues, please follow these troubleshooting steps:
Check the connection: Ensure the USB cable is securely plugged into both your computer and the Aha device.
Port Selection: Verify that you have selected the correct COM port in the application settings.
Browser: try a different browser such as Brave or Chrome!

*4. Will the app work on smartphone?*
Unfortunately not!

*5. Why the need for another app?*
The original software was created in Xojo. Xojo is an application builder, but it’s a very poor one since it simply performs very poorly on the computer (i.e., it crashes and is extremely slow). The official software automatically connects to api.ahaharmony.com and then downloads an .sqlite file (a file protected by a very strong password that we can’t access). That .sqlite file contains all the frequencies. What we did is we checked the logs generated by the official software, reverse engineered them, and intercepted all USB connections the computer makes with the Aha device. This way, we can read and write data.

*8. What are the specifications of Aha?*
Interface: Serial connection via USB (often using a CH340 or CP210x chip). Baud rate: 115200 bps. Command structure: Uses ASCII strings that begin with a colon (:) and end with a period (.) or a comma (,), followed by a Newline (\r\n). Aha has temporary buffer memory (RAM) for on-screen display and permanent Flash memory for storage. Write Protection: Crucial is the command :w09=0 to disable write protection and :w09=1 to re-enable it. Without this step, storage is not permanent. Aha has a EEPROM. Burning: The command :w84=255 activates the physical writing process to the chip, which takes about 3 to 4 seconds. Naming (:n): ASCII text linked to the slot number. Configuration (:w25): Expects exactly 10 parameters (Slot index + 9 technical parameters such as time and intensity). Frequencies (:w19): A sequence of numbers that must always end with a specific checksum (such as -4656). If the checksum does not match the frequencies, the hardware blocks the Play button.

*12. How does Aha calculate the frequencies?*
Let's take a look at the Cell program of Aha (in Spooky2 this is called Cell regeneration (BIO): The ratio between the Spooky2 frequency (97.5) and the value written to the Aha (9755) is based on a fixed scaling of 100 plus a correction offset. In Spooky2 program: 97.5 and in Aha: (:w19): 9755. The calculation performed by the official software is: (Frequency * 100) + 5. In this example: 97.5 x 100 = 9750. 9750 + 5 = 9755. During the data transfer I was watching, I saw :w19=9755,-756,.. The number -756 is the encrypted representation of the Dwell (duration). In Spooky2, the dwell is 180 seconds. Aha software uses a negative value or a specific bit shift to pass the duration per frequency to the hardware. In this case, -756 in the Aha firmware corresponds to the instruction to hold this frequency for exactly 180 seconds. Why the "-756"?
